# 非遗数据管理技能

## 概述
本技能帮助搜索互联网上的河南非物质文化遗产信息并将其插入数据库，与现有的Spring Boot + Vue3系统集成。它专注于网络数据采集、数据收集和数据库插入等遗产相关内容的处理。

## 核心功能

### 1. 网络搜索和数据收集
- 搜索非遗项目、传承人、活动和新闻
- 收集真实可用的多媒体资源，包括图片和视频
- 从搜索结果中提取结构化数据
- 验证和处理收集的信息
- 验证媒体资源的可用性和真实性

### 2. 数据库集成
- 将搜索结果插入MySQL数据库(heritage_db)
- 更新现有记录的新信息
- 处理媒体文件引用和存储
- 确保数据一致性和完整性
- 将媒体资源与非遗项目关联

### 3. 内容类型
- 带有真实图片和视频URL的非遗项目
- 传承人信息和简介
- 文化新闻和活动
- 相关媒体资源(图片、视频)

### 4. 媒体资源管理
- 搜索与非遗项目相关的高质量图片
- 从 reputable sources找到真实的视频资源
- 验证媒体资源的可访问性
- 优化媒体URL以进行数据库存储
- 确保媒体资源与非遗项目相关

## 使用说明

### 搜索带有媒体资源的非遗项目
1. 定义与河南非物质文化遗产相关的特定搜索关键词(例如："河南 少林功夫 非物质文化遗产 图片 视频")
2. 使用WebSearch工具查找关于该遗产项目的全面信息
3. 提取包括标题、描述、起源和意义在内的结构化数据
4. 搜索与该遗产项目相关的高质量、真实图片
5. 从博物馆、文化机构或官方媒体等来源找到官方或权威的视频资源
6. 通过检查URL验证媒体资源的可访问性
7. 验证和清理收集的数据
8. 将带有真实图片和视频URL的数据插入heritage_item表

### 搜索传承人
1. 使用特定关键词搜索河南的非遗传承人
2. 收集详细信息，包括姓名、性别、专业和成就
3. 从权威来源找到传承人的照片
4. 提取和处理数据
5. 将数据插入inheritor表

### 搜索文化新闻和活动
1. 搜索与河南遗产相关的近期文化活动和新闻
2. 收集活动详情，包括标题、日期、地点和描述
3. 从官方来源找到活动照片或视频
4. 对内容进行分类(新闻、活动、公告)
5. 将带有相关媒体引用的数据插入culture_news表

### 收集和验证媒体资源
1. 搜索与非遗项目相关的高质量图片和视频
2. 验证媒体URL是否可访问且来自权威来源
3. 检查媒体与特定非遗项目的相关性
4. 处理和优化媒体引用
5. 将媒体信息插入相关表(heritage_image, heritage_media)
6. 将媒体资源与相应的非遗项目关联

## API端点

### 非遗项目
- GET /api/heritage/list - 获取非遗项目列表
- POST /api/heritage/save - 保存新的非遗项目
- POST /api/heritage/update - 更新非遗项目
- DELETE /api/heritage/delete/{id} - 删除非遗项目

### 传承人
- GET /api/inheritor/list - 获取传承人列表
- POST /api/inheritor/save - 保存新的传承人
- POST /api/inheritor/update - 更新传承人
- DELETE /api/inheritor/delete/{id} - 删除传承人

### 文化新闻
- GET /api/culture-news/list - 获取新闻列表
- POST /api/culture-news/save - 保存新的新闻/活动
- POST /api/culture-news/update - 更新新闻/活动
- DELETE /api/culture-news/delete/{id} - 删除新闻/活动

### 评论
- GET /api/comment/listByHeritage - 获取非遗项目的评论列表
- POST /api/comment/add - 添加新评论
- DELETE /api/comment/delete/{id} - 删除评论

## 数据库结构

### 主要表
- heritage_item - 主要非遗项目信息
- inheritor - 非遗传承人信息
- culture_news - 文化新闻和活动
- heritage_comment - 用户评论
- heritage_category - 非遗分类
- region - 地理区域



## 最佳实践

1. **搜索策略**: 使用特定关键词获取更好的搜索结果(例如："河南 少林功夫 非物质文化遗产 图片 视频")
2. **媒体验证**: 在插入前验证媒体URL是否可访问且来自权威来源
3. **数据验证**: 在数据库插入前验证和清理搜索结果
4. **媒体质量**: 优先选择来自官方来源的高分辨率图片和高质量视频
5. **重复检测**: 检查现有记录以避免重复
6. **错误处理**: 为搜索和插入过程实现强大的错误处理
7. **速率限制**: 尊重搜索引擎的速率限制以避免IP被阻止
8. **数据归因**: 使用网络来源内容时正确归因
9. **性能优化**: 使用高效的搜索查询和批量数据库操作
10. **媒体相关性**: 确保媒体资源与特定非遗项目直接相关
11. **URL持久性**: 优先选择永久URL而非临时或基于会话的链接
12. **交叉验证**: 从多个来源交叉引用媒体资源以确保真实性

## 故障排除

### 常见问题
- **搜索API限制**: 搜索引擎可能会阻止过多的请求
- **数据提取**: 从非结构化网络内容中提取结构化数据的困难
- **媒体验证**: 媒体URL不可访问或来自不可靠来源
- **数据库插入**: 由于验证规则导致的数据插入错误
- **网络连接**: 搜索操作期间的网络连接问题
- **媒体相关性**: 与非遗项目无直接关联的媒体资源
- **URL持久性**: 可能过期的临时或基于会话的链接

### 调试技巧
1. 检查搜索引擎响应代码和错误消息
2. 在数据库插入前验证提取的数据结构
3. 在数据库插入前测试媒体URL的可访问性
4. 验证数据库表约束和外键关系
5. 监控网络连接并重试失败的搜索请求
6. 检查后端日志中的API错误和数据库异常
7. 从多个来源交叉引用媒体资源
8. 定期测试媒体URL以确保它们保持可访问
9. 尽可能使用URL缩短器或永久链接
10. 验证媒体文件格式是否被前端支持

## 结论
本技能提供了全面的功能，用于搜索互联网上的河南非物质文化遗产信息并将其插入数据库，支持文化遗产的数字保护和推广。通过自动化数据收集和数据库插入过程，它实现了更高效、更全面的遗产信息管理，包括项目、传承人、活动和媒体资源。本技能帮助确保有价值的文化遗产信息被系统地收集、组织并通过数字平台提供访问。
