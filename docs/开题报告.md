# 河南非遗数字展示系统设计与实现

## 1. 选题背景及意义

### 1.1 选题背景
非物质文化遗产（以下简称“非遗”）是中华民族优秀传统文化的重要组成部分，是人类文明的瑰宝。河南作为中华文明的重要发祥地，拥有丰富的非遗资源，如豫剧、少林功夫、钧瓷烧制技艺、汴绣等，这些都是中华民族宝贵的文化遗产。然而，随着现代化进程的加速，许多非遗项目面临着传承困难、传播渠道单一、保护手段落后等问题。

传统的非遗展示方式主要依赖于博物馆、展览等线下形式，存在着受众范围有限、展示形式单一、互动性差等不足。在数字化时代，如何利用现代信息技术手段，创新非遗保护和传承方式，成为当前非遗工作的重要课题。

同时，随着互联网技术的快速发展，特别是Web技术的成熟，为非遗的数字化展示提供了新的机遇。通过构建数字化展示系统，可以突破时间和空间的限制，让更多人了解和接触非遗文化，为非遗的保护和传承注入新的活力。

### 1.2 选题意义
#### 1.2.1 文化价值
- **保护传承**：通过数字化手段，对河南非遗资源进行系统整理和展示，有助于非遗文化的保护和传承
- **文化传播**：扩大非遗文化的传播范围，提高社会对非遗的认知度和关注度
- **文化自信**：展示河南丰富的非遗资源，增强文化自信，促进文化认同

#### 1.2.2 社会价值
- **公众教育**：为公众提供了解非遗文化的平台，促进非遗知识的普及
- **旅游推广**：结合河南旅游资源，推动文化旅游融合发展
- **社区参与**：鼓励公众参与非遗保护，形成全社会共同保护非遗的良好氛围

#### 1.2.3 技术价值
- **技术应用**：将现代Web技术应用于非遗保护领域，探索数字化保护的新途径
- **系统集成**：整合前后端技术，构建功能完整的数字化展示系统
- **实践创新**：通过项目实践，提高软件开发能力和技术创新能力

## 2. 设计目的、要求、思路与预期成果

### 2.1 设计目的
随着数字化时代的到来，非物质文化遗产的保护和传承面临新的挑战与机遇。河南作为中华文明的重要发祥地，拥有丰富的非物质文化遗产资源。然而，传统的展示方式难以满足现代社会对非遗文化的传播需求。本项目旨在开发一个河南非遗数字展示系统，通过数字化手段实现非遗资源的有效管理、展示和传播，提高非遗文化的社会认知度和影响力，为非遗的保护和传承提供新的途径。同时，通过系统开发，学习和应用现代Web技术，提高软件开发能力和实践应用水平。

### 2.2 设计要求
- 系统设计方案合理，符合非遗展示的实际需求
- 系统功能完整，包括非遗项目、传承人、宣传、活动等核心模块
- 代码规范，分层逻辑清晰，具有良好的可读性和可维护性
- 运行可靠，数据处理正确，实现数据的增删改查等基本功能
- 用户界面友好，使用方便，具有良好的视觉效果和用户体验
- 能够在社会、文化、技术等现实约束条件下，对系统进行分析、评价和优化

### 2.3 设计思路
该系统将采用前后端分离的架构，后端使用Spring Boot + MyBatis + MySQL构建RESTful API，前端使用Vue3 + Vite + Element Plus构建用户界面。系统将实现非遗项目的数字化管理、传承人信息的展示、宣传活动的发布、用户互动等功能，通过现代化的Web技术为用户提供沉浸式的非遗文化体验。

### 2.4 预期成果
- 完成一个功能完整、界面美观的河南非遗数字展示系统
- 实现非遗项目、传承人、宣传、活动等核心模块的管理和展示
- 提供用户友好的界面，满足不同用户群体的使用需求
- 为非遗文化的数字化保护和传播提供有效工具
- 提高团队的软件开发能力和项目管理经验
