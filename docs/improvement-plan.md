# 河南非物质文化遗产数字化展示系统 - 改进计划

## 1. 项目现状分析

### 1.1 现有功能
- 非遗项目展示
- 传承人展示
- 文化资讯
- 非遗宣传（图片+视频）
- 用户收藏
- 用户评论
- 搜索功能
- 地图展示（河南省非遗项目分布）
- 后台管理（用户、非遗项目、分类、资讯、评论、图片、视频管理）

### 1.2 技术栈
- 后端：Spring Boot + MySQL
- 前端：Vue3 + Vite + Element Plus
- 数据可视化：ECharts

### 1.3 现有问题
- 多媒体展示功能已删除，缺少非遗项目的视觉呈现
- 地图功能虽已实现，但交互性和功能完整性有待提升
- 用户互动功能相对简单，缺乏深度
- 搜索和筛选功能需要优化
- 内容深度和丰富度需要加强
- 技术亮点不够突出，难以在毕业设计中脱颖而出

## 2. 核心改进建议

### 2.1 功能改进
1. **重新整合多媒体展示**
   - 在非遗项目详情页添加图片轮播和视频展示
   - 实现图片点击放大查看功能
   - 为多媒体内容添加说明文字，增强文化内涵

2. **地图功能优化**
   - 实现地图点击事件，点击地区后筛选显示该地区的非遗项目
   - 添加地图图例说明，帮助用户理解热力图的含义
   - 优化悬停效果，显示更详细的地区非遗信息
   - 调整地图配色方案，使用更符合传统文化氛围的色彩

3. **用户互动增强**
   - 完善用户收藏功能，增加收藏夹管理
   - 增强评论系统，支持回复和点赞功能
   - 添加用户贡献功能，允许用户上传与非遗相关的内容（需审核）

4. **搜索与筛选优化**
   - 实现关键词联想和模糊搜索
   - 添加多维度筛选条件（项目类型、级别、地区、年代等）
   - 实现筛选结果的实时更新和排序功能

5. **内容深度提升**
   - 为每个非遗项目添加详细的历史背景和传承情况
   - 增加传承人故事和访谈内容
   - 添加非遗项目的制作工艺流程介绍
   - 整合相关历史文献和研究资料

### 2.2 技术亮点
1. **智能推荐系统**
   - 基于用户浏览历史和收藏记录，推荐相关非遗项目
   - 实现简单的协同过滤算法，提升推荐准确性



3. **性能优化**
   - 实现图片懒加载和资源预加载
   - 优化数据库查询，减少响应时间
   - 利用缓存机制提升系统性能




## 4. 技术实现方案

### 4.1 前端实现
- **多媒体展示**：使用 Element Plus 的 `el-carousel` 组件实现图片轮播，集成 `viewerjs` 实现图片放大查看
- **地图交互**：使用 ECharts 的地图点击事件，实现与列表的联动
- **智能推荐**：使用 Vue3 的 `reactive` 和 `watch` 实现状态管理，结合 LocalStorage 存储用户行为数据
- **响应式设计**：优化现有响应式布局，确保在不同屏幕尺寸下的显示效果

### 4.2 后端实现
- **API 优化**：优化现有 API 接口，添加新的功能接口
- **数据库设计**：扩展数据库表结构，支持新功能的数据存储
- **缓存机制**：使用 Spring Boot 的缓存机制提升性能
- **数据统计**：实现数据统计功能，支持后台管理的数据可视化

### 4.3 数据管理
- **数据收集**：从公开渠道收集河南非遗项目的详细信息
- **数据整合**：整合多源数据，确保数据的准确性和完整性
- **数据维护**：开发后台管理功能，方便数据的更新和维护
